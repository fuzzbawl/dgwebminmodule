dossier Paths
  
Le chemin complet vers un fichier (par exemple /usr/local/etc/dansguardian/fichier).

Dans la plupart des cas, aucun de ces chemins doivent être changées. Tous les paramètres par défaut de suivre le même schéma et, en général, ne doivent pas être annulés individuellement.

(Les fichiers sous /usr/share sont des copies des originaux. Si un fichier est irrémédiablement gâché, vous pouvez copier l'un de ces plus et commencez à tous. Ils ont bien sûr le même nom que le les fichiers que vous avez l'habitude de voir et d'édition. Ces fichiers sont utilisés pas à tous par un système fonctionnant dansguardian. Comme ces fichiers ne sont que de la récupération, ne changent pas leur.)

La plupart des chemins de fichiers dansguardian commencer par un «préfixe» qui en est de même pour presque tous les chemins qui instance de dansguardian. Le préfixe est généralement soit /usr/local ou "" (rien), l'origine du défaut, par exemple pour être soit /usr/local/etc/dansguardian/fichier ou /etc/dansguardian/fichier.

Le chemin est défini lorsque le préfixe est construit dansguardian (le 'configure' est nommé paramètre - prefix). Il ne peut pas être changée dansguardian sans recompilation. Vous pouvez déterminer ce qui a été construit en préfixe votre instance de l'exécution par dansguardian dansguardian -v (le paramètre mai pas si elle est fixée à /usr/local).

    Le code HTML
