afficher ou modifier l'échelle du système de configuration
  
  

Afficher ou modifier les paramètres de la plupart de l'échelle du système dansguardian options de configuration. Ce sont les paramètres qui sont (ou pourraient être) dans le fichier de configuration dansguardian.conf. Si vous souhaitez afficher ou modifier les paramètres de chacun des groupes de filtre (les options qui sont [ou qui pourraient être] dans le dansguardianfN.conf des fichiers de configuration) utiliser la fonction "Afficher/Modifier un filtre du Groupe de la Base de Config" outil. Si vous souhaitez afficher ou modifier une liste de configuration, utiliser la fonction "Afficher/Modifier un filtre du Groupe Listes" outil (ou dans quelques cas, la "Afficher/modifier l'échelle du système des Listes" outil).

La disposition l'ensemble du système des options de configuration sont regroupées dans les différents "onglets" dans cette interface utilisateur. De cette façon, vous pouvez manipuler un groupe d'options tout ne change pas (ou même à la recherche) d'autres options. (Si vous connaissez le nom de l'option que vous souhaitez manipuler, mais je ne sais pas qui c'est sur l'onglet, utiliser l'onglet en haut finder pour aller à l'onglet à droite automatiquement.)

Le réglage actif est indiqué avec une bordure autour de lui. Comme on fait les modifications proposées, la frontière mai déplacer.

Pour chaque option, le "Set?" case à cocher indique si oui ou non certains paramètre de l'option est présente dans le fichier de configuration. (Ce qui permet le cas des valeurs vides tels que 0 ou 'off' et de l'absence totale d'un cahier des charges de l'option Il convient de distinguer sans ambiguïté.) Pour changer ou non d'une option de paramètre est présent, cliquez sur la case à cocher.

Si une option de réglage est (ou sera) actuelle, le "System-Wide Value" zone va montrer sa valeur. Le "System-Wide Value" à proximité de la zone de gauche, ont une bordure autour de lui afin que vous puissiez plus facilement discerner quelles sont les options qui sont (ou seront) présent. Si, en revanche, l'option? est pas est cochée, il sera fait référence à une valeur par défaut si il y en a un. Dans la plupart des cas, la frontière se déplace à la "carence" espace à la droite. Vous ne pouvez pas changer les valeurs par défaut, mais vous pouvez lire ce qu'ils sont.

(La "par défaut" est la valeur compilée en valeur qui seront réellement utilisés si l'option n'est pas précisée. Il est souvent pas, la valeur recommandée ni celui qui apparaît dans les fichiers de configuration par défaut.)

Sur le "reporting" tab, la 'htmltemplate' option ne sera utilisée si le 'reportinglevel' option est d'utiliser HTML template file. De même, le 'accessdeniedaddress' et 'nonstandarddelimiter' options ne seront pas utilisées à si le 'reportinglevel' option n'est pas mis à utiliser le HTML template file. Filtre individuel peuvent l'emporter sur toutes les options sur le "reporting" tab, options afin mai en fait être utilisés ailleurs même si cet affichage indique qu'ils sont inutilisés.
