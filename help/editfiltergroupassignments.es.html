Filtro Grupo de Tareas
  

Una vez que los usuarios se identifican, pueden ser asignados filtro a los diferentes grupos. Cada filtro tiene su propio grupo de configuración. El filtro de múltiples Grupos característica es como correr varios copias de DansGuardian en paralelo de una sola vez, inteligente y dinámicamente la asignación de cada usuario el derecho a una.

Los usuarios pueden ser identificados tanto por la persona por el nombre de usuario o dirección IP del equipo. (La mayoría de los métodos que identifican a las personas por nombre de usuario algunos también requieren la asistencia de la representación local [Calamar?].) Normalmente los usuarios identificar las instalaciones, ya sea por nombre de usuario o por equipo de propiedad intelectual, no una combinación. De hecho, en muchos casos, un solo authplugin de cualquier tipo está habilitado.

Después de los usuarios están identificados, dos listas de control la cartografía de los usuarios para filtrar grupos. Uno de ellos (.../ listas / filtergroupslist) se utiliza para la identificación por nombre de usuario, y, en muchos casos, controla la mayoría o la totalidad de la cesión. Las otras listas (.../ / authplugins / ipgroups) se utiliza para la identificación por ordenador la dirección IP.

La sintaxis general de ambos archivos es la misma: identificador=filter1-99 1-99, donde están los grupos de filtro. Para la identificación por el usuario, identificador es el nombre de usuario, un ejemplo de modo de entrada puede ser: 

danielg=filter2
Para la identificación por computadora, identificador es una dirección IP, un rango de direcciones IP, o una dirección IP de subred, tan ejemplos de entradas podrían ser: 
192.168.0.100=filter1
192.168.0.200-192.168.0.230=filter2
192.128.1.0/24=filter3
