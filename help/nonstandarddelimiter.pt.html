Uso Não-padrão Delimitador Quando Constructing URLs para a CGI Reporting Programa
  

A não-padrão delimitador (algo diferente do habitual & e =) só será usada em conjunto com accessdeniedaddress. Por isso, não terá efeito se reportinglevel é 3.

Para facilitar a plena incorporação proibido URL no URL que é passado para accessdeniedaddress sem causar confusão, o delimitador dentro da consulta parte do URL pode usar as duas seqüências de caracteres :: (em vez de & == (em vez de =. Seu dansguardian.pl-como script CGI irá naturalmente têm de analisar a consulta parte do URL utilizando estes suplentes delimitadores.

Vire sobre este caso requer que o script CGI ou achar que torna mais fácil codificação seu script CGI. Nota ainda que, embora este resolve o problema da confusão e pode ser compatível com a maioria dos programas CGI, É totalmente não-padrão e não será suportado por utilidade subrotinas ou bibliotecas.
