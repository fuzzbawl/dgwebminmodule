Conteúdo Scanner plug-ins
  

Estes conteúdos varredura plugins fornecer capacidades para a realização de diversos tipos de conteúdo adicional varredura antes DansGuardian libera um arquivo para um usuário. Muitas vezes, tudo o que é necessário para activar / desactivar (descomentar / comentário) individuais queridos, que pode ser feito através do "plugins" guia nos principais config edição tarefa janela. Eles são usados principalmente para anti-vírus digitalização.

Será que não permitir ambosclamav e clamdscan, ao mesmo tempo. Eles fornecem a mesma funcionalidade, tanto assim que apenas usar-se um lote adicional de recursos da CPU sem fornecer uma melhor proteção contra vírus. O clamdscan método é mais flexível e é preferida em todos os casos. O clamav método - que também pode complicar edifício -- está presente principalmente para compatibilidade com versões anteriores. (A - enable-clamav opção é construir não necessário utilizar clamdscan; apenas o - enable-clamd construir opção é necessário.)

Se quiser implementar um scanner que não é incluída na lista de opções pré-fornecidos (por exemplo, alguns outro tipo de anti-virus, ou algum outro tipo de verificação inteiramente tais como um que tenta distinguir imagens pornográficas), use o comando escolha.

Cada plugin tem uma configuração própria. Em muitos casos, essas configurações não precisam ser alterados, Se uma das configs não necessitam de ser alterados, você pode acessar o arquivo de configuração para a edição através do plugin config edição ferramenta.

Algumas das opções podem dizer #! Não compilado! . Se assim for, não podemos jamais permitir que config combinação qualquer destas opções em seu sistema. Para fazer essas opções disponíveis, você deve obter tanto um executável DansGuardian que inclui suporte para a opção e config que igualar. Uma maneira de fazer isso é reconstruir DansGuardian da fonte. Reconstruindo inclui executando. / Configure, compilação, ligação e instalação; não deve ser empreendida de ânimo leve.
