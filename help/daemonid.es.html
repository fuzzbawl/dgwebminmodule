demonio del usuario / grupo
  
Al igual que todos los procesos, DansGuardian el servidor / demonio proceso tiene un usuario y grupo. El usuario o grupo de cada proceso controla cómo el sistema operativo que le otorga los permisos, en particular el acceso a los distintos archivos y directorios. Normalmente la operación es más sencilla si DansGuardian, su representante local de backend (por ejemplo, calamar), y cualquier antivirus demonio todos se ejecutan como el mismo usuario / grupo

Si no se especifica, por defecto se DansGuardian a ejecutar como cualquier usuario y de grupo fueron construidos en (por lo general bien nadie / nadie o nouser / nogroup). Es probablemente mejor explititly especificar algún usuario userid / group oportunidad para que los permisos de los archivos según sea necesario para que el construido en los entornos de trabajo.
