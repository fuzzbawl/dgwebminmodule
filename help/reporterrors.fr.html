Reporting "Access Denied" Pour les utilisateurs
  

Dansguardian a deux méthodes différentes pour rendre l'accès est refusé aux utilisateurs. (Ces paramètres sont de présenter un "accès refusé" à la page des utilisateurs. Ils n'ont pas d'effet sur ce qui se passe dans le journal ou comment.)

Dans la plupart des cas, vous devez utiliser la plus récente et la plus rapide et plus propre HTML File méthode. Il n'a pas besoin d'un serveur web ou la mise en place d'un programme CGI, et il est plus facile de personnaliser la page d'accès refusé. Set reportinglevel à 3, permettre accessdeniedaddress pour être ignorée, et laissez htmltemplate par défaut à son habitude PREFIX / languagedir / langue / template.html.

Utilisez l'ancienne méthode de pointage accessdeniedaddress à votre propre installation d'un "dansguardian.pl"-like script sur votre propre serveur web seulement dans des situations inhabituelles des circonstances particulières dans lesquelles vous avez besoin d'un maximum de flexibilité.

Bien que par défaut, tous les groupes d'utiliser le filtre de mêmes paramètres rapport à l'accès refusé, il est possible pour chaque filtre de groupe à avoir ses propres réglages. Par exemple, il est possible de faire filtrer les différents groupes utiliser différents htmltemplate des fichiers, y compris certains un "bypass" installation et d'autres pas. Dans le cas habituel dans lequel remplace ne sont pas nécessaires, vient préciser les paramètres une fois dans dansguardian.conf. Pour fournir les différents paramètres de filtre pour les différents groupes, préciser les paramètres de l'individu dansguardianf N. conf fichiers.

Les paramètres sont:

    Niveau
  • Reporting
    • -1 = journal, mais ne pas bloquer (mode furtif)
    • 0 = Juste dire "Accès refusé"
    • 1 = rapport, mais pourquoi ne pas nier ce que la phrase
    • 2 = Rapport pleinement
    • 3 = Utilisez le modèle de fichier HTML (Access Denied Adresse sera ignorée)
    La valeur par défaut est 3, qui précise en utilisant un fichier HTML (plutôt que d'un programme CGI) et est généralement le cas.

  • HTML Fichier modèle Path
    Seulement utile lorsque reportinglevel est fixée à 3 et filtre certains groupes ont leur propre page d'accès refusé. Permet le fichier HTML à lire sur un autre emplacement plutôt que de l'habituel PREFIX / languagedir / langue / template.html.
    Si spécifié, ce doit être un chemin complet à partir de "/". Bien que partielle des chemins mai cause pas une erreur de syntaxe dans tous les cas, à la fin, ils ne fonctionnent pas à droite.

  • Access Denied Adresse
    Utilisé uniquement lorsque reportinglevel est pas set à 3, ignoré sinon. L'adresse de votre déclaration script CGI qui vous avez installé et l'exécution sur votre propre serveur Web. Votre serveur web n'a aucun lien avec dansguardian autres que celles prévues par cette URL. Ainsi, votre serveur web mai mai ou ne pas être en cours d'exécution sur sur le même ordinateur que votre dansguardian.
    Cela doit être une URL commençant par "http://". Bien que partielle URL mai de ne pas provoquer une erreur de syntaxe dans tous les cas, à la fin, ils ne fonctionnent pas à droite.
