Auth plug-ins
  

Estes auth plugins fornecer vários tipos de usuário (ou computador) autenticação ou autorização. Normalmente tudo o que é necessário para activar / desactivar (descomentar / comentário) deles, que pode ser feito através do "plugins" guia nos principais config edição tarefa janela. (A ordem da sua lista é importante. Mas o arquivo de configuração já enumera esses plugins na única forma razoável, assim não há possibilidade de alterar a ordem é incluído aqui.)

Auth métodos podem ser categorizadas como segue:

  • identificar usuário
    • necessitar de assistência do local proxy (Lula?)
      (estas opções serão nunca, em um trabalho completamente "transparente-interceptor" ambiente)
      • Básicas longa web externo padrão
      • Digest mais seguras desafio-resposta padrão W3C, pode não ser totalmente suportados em todos os lugares
      • NTLM da Microsoft maneira
    • não exigem qualquer assistência
      • ident, o mais fácil spoofable
  • identificar computador
    • IP
Cada plugin tem uma configuração própria. Na maioria dos casos, estas configs nunca devem ser alterados a todos, (A única exceção é em alguns casos detalhados edição do ipgroups que o arquivo da filial para o IP auth plugin será necessária.) Se uma das configs não necessitam de ser alterados, você pode acessar o arquivo de configuração para a edição através do plugin config edição ferramenta.

Algumas das opções podem dizer #! Não compilado! . Se assim for, não podemos jamais permitir que config combinação qualquer destas opções em seu sistema. Para fazer essas opções disponíveis, você deve obter tanto um executável DansGuardian que inclui suporte para a opção e config que igualar. Uma maneira de fazer isso é reconstruir DansGuardian da fonte. Reconstruindo inclui executando. / Configure, compilação, ligação e instalação; não deve ser empreendida de ânimo leve.
