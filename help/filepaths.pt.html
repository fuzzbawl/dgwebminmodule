Arquivo Caminhos
  
O caminho completo para um ficheiro (por exemplo / usr / local / etc / dansguardian / arquivo).

Na maioria dos casos, nenhum dos caminhos deve ser mudado. Todos os padrões seguem o mesmo padrão e, geralmente, não deve ser ultrapassado individualmente.

(Os arquivos sob / usr / share são extra cópias dos originais. Se um arquivo fica irremediavelmente confuso, você pode copiar um destes e começar tudo. Eles evidentemente têm os mesmos nomes como o arquivos que você está acostumado a ver e editar. Esses arquivos são usados não a todos por um sistema executando o DansGuardian. Como esses arquivos são somente para a recuperação, não alterá-los.)

A maioria dos caminhos DansGuardian arquivo começa com um "prefixo" esse é o mesmo para quase todos os caminhos em que instância do DansGuardian. O prefixo é normalmente quer / usr / local ou "" (nada), causando o exemplo padrão para ser / usr / local / etc / dansguardian / arquivo ou / etc / dansguardian / arquivo.

O caminho está definido quando prefixo DansGuardian é construída (o 'configure' parâmetro é chamado - prefix). É não pode ser alterado sem recompilar DansGuardian. Você pode determinar qual o prefixo foi construído em a instância do DansGuardian executando dansguardian -v (o parâmetro não pode aparecer se ele é definido como / usr / local).
