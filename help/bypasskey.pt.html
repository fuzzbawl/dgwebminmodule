temporária negado / Infected Page Bypass chave secreta
  

Bypass funcionalidade utiliza uma chave secreta hashed para proporcionar segurança.

Com o nível 3 e no modelo HTML, máximo de segurança é fornecida através de um diferente hashed chave secreta de cada vez. Este comportamento de escolher uma nova chave de cada vez é especificado pela omitir esta opção (ou indicando-o como uma cadeia vazia).

Com o nível superior a 3 valores e outros programas CGI que deve gerar o bypass URL, essa randomização não irá funcionar. Assim, é possível, em vez de gerar uma chave aleatória de cada vez para indicar explicitamente uma chave para ser usada de cada vez. Deve ser superior a 8 caracteres, por exemplo
bypasskey   =   "Mary tinha um pouco de cordeiro '. ou bypasskey   =   '76b42abc1cd0fdcaf6e943dcbc93b826 '.

A mesma funcionalidade está prevista para as páginas que vírus são infectadas (em vez de negada por causa do conteúdo) pela configuração opção infectedbypasskey, com as mesmas configurações possíveis.
