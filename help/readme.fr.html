Module Webmin dansguardian Améliorations de la version précédente
  

La plupart des améliorations importantes

  • Log Analysis est considérablement élargie, indiquant notamment le détail des expressions régulières matches. En plus de produire plus finement filtré et des rapports plus détaillés, Log Analysis travaille maintenant dans un couple des environnements qui ne sont pas typiques pour les modules Webmin:.
    • Log Analysis peut être utilisé sur un ordinateur différent que celui où dansguardian production s'exécute. Vous pouvez copier les logs à partir de votre serveur de production vers un autre ordinateur, et d'analyser les journaux là-bas. Ainsi, l'analyse de logs ne doivent pas affecter les performances du serveur de production.
    • Aussi Log Analysis peut être automatiquement activé pour produire des rapports de lot sans surveillance. Le fait que Log Analysis est intégré dans Webmin ne pas se limiter à une utilisation interactive.
  • L'accent est mis sur «groupe de filtres multiples configurations. Dansguardian la capacité de traiter différemment les différents utilisateurs est un élément central de cet outil de gestion. Il peut aussi bien créer des fichiers de configuration et de modifier les fichiers de configuration d'une manière qui est très conscient des filtres multiples groupes.
  • Tant l'activation et la configuration de tous les Dansguardian «plugins» sont entièrement pris en charge par ce module. Les plugins peuvent être mis en place pour fournir des différentes saveurs de "auth", auxiliaires l'analyse de contenu tels que les outils anti-virus, et de télécharger des améliorations.

Requirements

Contrairement aux versions précédentes de ce module ou la plupart des autres modules Webmin, Ce module utilise JavaScript fortement. En fait, vous n'aurez probablement pas même être en mesure d'utiliser ce module Webmin si vous désactivez JavaScript dans votre navigateur.

Cette version de ce module se concentre sur convivialité au détriment d'une augmentation substantielle des exigences de CPU et de RAM. À condition que ce module est exécuté sur un ordinateur moderne avec adéquate CPU et RAM, tout fonctionne bien. Mais ce module ne sera probablement pas accomplir de façon adéquate sur un ancien petit ordinateur lent, même si elle a les versions précédentes de ce module, okay.

Il semble qu'un malaise traduction serait particulièrement utile, mais malheureusement il n'est pas encore disponible. Si une traduction partielle malais ont contribué, il sera utilisé là où il était disponible, et la norme Webmin mécanisme automatiquement à une autre langue en d'autres lieux.

Recherche détaillée Enhancements

La recherche est maintenant de son propre outil plutôt que d'être greffé sur une fonction d'un autre outil.

Une grande variété de choses peuvent être recherchées pour intelligemment: mots, de phrases, d'un domaine (d'accueil) le nom, l'URL, et même modèle. On peut chercher et trouver une expression régulière simplement en tapant l'expression littérale. Il n'y a pas besoin d'utiliser un étrange syntaxe pour la recherche, pas simplement à traduire certains caractères. Recherche d'une phrase prend automatiquement en compte éventuellement différente de l'ordre des mots. Taper juste "Jack sauté" sera automatiquement recherche pour "sauté Jack" aussi.

Pour chaque fichier dans lequel un ou plusieurs matches a été trouvée, si oui ou non le fichier de filtre s'applique à chaque groupe seront présentés. Ainsi, si le contenu d'un fichier s'appliquent à certains groupes de filtre mais pas d'autres, il sera immédiatement clair. Filtrer les groupes sont classés par nom de groupe de filtre (si le groupe a un nom), ce qui rend plus facile à comprendre l'affichage seulement par les regards jetés à elle.

Enseignement Enhancements

Têtes et pieds de page sont plus cohérentes, comme l'affichage de messages d'alerte. Tant la version dansguardian et le module Webmin version sont affichés sur chaque page, et toujours au même endroit.

Test de l'environnement et l'affichage de messages d'alerte si quelque chose ne semble pas juste ont été considérablement élargies.

Tout a changé dans la ligne de chaque fichier est commenté avec la date et l'horodatage indiquant quand il a été changé.

Insertion d'options dans les fichiers de configuration est semi-intelligente, et les nouvelles options ne sont pas tout juste pile à la fin du fichier. Si l'option existe déjà mais a ensuite été commentée et maintenant, est ajouté le dos, la nouvelle option sera immédiatement suivi l'ancien commentaire n'importe où dans le fichier.

Chaque fois que des filtres multiples groupes existent, filtre tous les groupes sont inscrits à côté de l'autre (souvent en onglets sur la même page). Et ils sont souvent appelées par leur nom attribué plutôt que de 'f1', 'f2', 'f3', et ainsi de suite.

Pour modifier le fichier de configuration, options de configuration sont réparties sur un grand nombre d'onglets de menu. Pour les moments où vous connaissez le nom de l'option vous voulez modifier mais je ne sais pas qui c'est sur l'onglet, il ya une option qui énumère chercheur toutes les options et saute immédiatement à l'onglet à droite lorsque vous sélectionnez un.
