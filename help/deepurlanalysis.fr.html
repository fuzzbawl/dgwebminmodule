Lorsque cette option est activée, dansguardian ressemble l'intérieur d'autres adresses URL pour les URL et les contrôles à l'intérieur contre les URL bannedsitelist (et exceptionsitelist) et bannedurllist (et exceptionurllist). Ceci est particulièrement utile pour bloquer les images provenant de sites interdits d'apparaître dans les résultats de recherche Google Images, que l'original URL sont intégrées dans les vignettes des requêtes GET.

L'utilité de cette option pour limiter l'utilisation de proxies anonymes est moins claire.

Cette option ne fait même pas essayer d'identifier "masquée" URLs, tels que ceux traités avec codage hexadécimal, l'encodage en base64, voire Rot13 encodage. Ces encodages d'URL, généralement effectuée avec Javascript, sont très fréquentes.

A noter qu'il ya aussi des raisons légitimes d'avoir une URL dans une autre URL, par exemple pour plus de sécurité, de précision, et de transmettre facilement "referrer" service d'information de sites Web. Si des problèmes sont détectés à cette option, envisager d'expérimenter avec juste de l'éteindre.
