Méthodes d'identification des utilisateurs
Vous pouvez définir autant de méthode que vous voulez, et pas juste une. La première sera utilisée, et si l'utilisateur n'est pas trouvé, la suivante est utilisée (et ainsi de suite).

Proxy Auth
Proxy Auth est utilisée pour une identification simple via le proxy (pas intéressante dans le cas d'un proxy transparent).

Ident Auth
Ident est utilisé dans le cas où toute autre méthode ne marche pas. DG va contacter la machine dont est issue la connexion et lui demander le possesseur de cette connexion. Un serveur Ident est nécessaire sur la machine en question.
Pour plus d'informations, voir Identd for Win9x pour un serveur Ident sur plate-forme Windows.
