Filtre Groupe Missions
  

Une fois que les utilisateurs sont identifiés, ils peuvent être assignés filtre à différents groupes. Chaque filtre groupe a sa propre configuration. Le filtres multiples Groupes comme la course est fonction de plusieurs des copies de dansguardian en parallèle à la fois, dynamiquement et intelligemment l'attribution à chaque utilisateur à la bonne.

Les utilisateurs peuvent être identifiés soit par la personne nom d'utilisateur ou par l'adresse IP de l'ordinateur. (La plupart des méthodes qui permettent d'identifier les personnes par nom d'utilisateur également besoin de l'aide de la proxy local [Squid?].) Généralement installations d'identifier les utilisateurs par nom d'utilisateur ou par IP, et non une combinaison. En fait, dans de nombreux cas, un seul authplugin de tout type est activée.

Lorsque les utilisateurs sont identifiés, deux listes de contrôle la cartographie des utilisateurs de filtrer les groupes. L'un d'eux (.../ lists / filtergroupslist) est utilisée pour l'identification par nom d'utilisateur, et dans de nombreux cas, la plupart ou la totalité des contrôles de l'affectation. Les autres listes (.../ / authplugins / ipgroups) est utilisée pour l'identification par adresse IP.

La syntaxe générale de ces fichiers est le même: identifiant=filter1-99 1-99, où sont le filtre groupes. Pour l'identification par l'utilisateur, identifiant est le nom d'utilisateur, donc un exemple d'entrée peut être: 

danielg=filter2
Pour l'identification par l'ordinateur, identifiant est une adresse IP, une plage d'adresses IP, adresse IP ou un sous-réseau, si par exemple les entrées peuvent être: 
192.168.0.100=filter1
192.168.0.200-192.168.0.230=filter2
192.128.1.0/24=filter3
